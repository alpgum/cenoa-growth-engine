# Performance Marketing Team OS (Cortex Module)

Purpose: a lightweight operating system that turns weekly data pulls + health checks into **clear decisions** (pause/scale/fix/ship) with explicit owners.

## 0) Definitions (so decisions are consistent)

- **Primary outcome (default):** `Activated` user (KYC passed + first key activation event). If the business defines a different “activation” (e.g., first transfer, card order, first top-up), replace the activation event but keep the OS.
- **Target CPA/CAC:** set monthly per geo/platform (owned by Acquisition + Product liaison). If targets are missing, decisions use relative performance (vs 7d avg) until targets are set.
- **Attribution source of truth (default):** AppsFlyer for paid; Amplitude/GA4 for product/web sanity checks.
- **Learning budget:** budget explicitly allowed to be “inefficient” for exploration (new geo/creative/audience). Everything else is efficiency budget.

## 1) Roles & Responsibilities

### 1.1 Acquisition Lead (Paid Growth)
Owns spend, structure, and performance.

- Own channel strategy and execution (Meta, Google, ASA, etc.).
- Own campaign/ad set/account structure (naming, segmentation, testing lanes).
- Own **budget allocation** across countries/platforms/testing lanes.
- Own **decision log**: pause/scale calls and why.
- Own weekly “next actions” list and ensures actions are shipped.
- Own vendor/platform comms for fraud/appeals/policy issues.

**Outputs:** budget plan, changes applied in-platform, action items, scaling plan.

### 1.2 Creative Lead (Performance Creative)
Owns creative throughput and quality aligned to performance signals.

- Own creative testing pipeline (brief → production → QA → trafficking → analysis).
- Own creative taxonomy (concept / angle / format / hook / CTA) so learning accumulates.
- Own weekly creative review and creative scorecards.
- Own “creative fatigue” plan (rotation, refresh, variations) per geo.
- Own landing/storefront creative alignment (LP, store listing, message match).

**Outputs:** creative briefs, production tracker, top/bottom concepts, next batch plan.

### 1.3 Data Lead (Marketing Analytics)
Owns data integrity, automation, and analysis that drives decisions.

- Own data pipelines & monitors:
  - weekly KPI pulls
  - anomaly detection
  - campaign health checks
  - budget pacing
  - data quality monitoring
- Own metric definitions + documentation (data dictionary).
- Own “single source of truth” files in `/data` and reporting in `/analysis`.
- Own incident triage for attribution breaks (and coordinates escalation).
- Own query/reconciliation for cross-system mismatches.

**Outputs:** weekly report, anomalies list, health status, data quality status, recommended actions with evidence.

### 1.4 Product Liaison (Growth PM / Eng Liaison)
Owns removing product-side blockers and ensuring marketing learns convert into product work.

- Own the feedback loop from paid traffic → onboarding/KYC/funnel fixes.
- Own prioritization for experiments that need product changes (LP changes, paywall, KYC flow, event instrumentation).
- Own escalation path into engineering/data for:
  - KYC bugs
  - tracking/instrumentation issues
  - release coordination for growth experiments
- Own “impact translation”: quantify funnel impact of product issues on CAC.

**Outputs:** product action tickets, escalation summaries, agreed KPI targets/constraints.

## 2) Weekly Cadence (non-negotiable meetings + async work)

### Monday: KPI / Anomaly / Health / Actions (60–90 min)
Goal: start the week with a single shared view of reality and a prioritized action plan.

**Pre-read (generated by Data Lead by 09:30):**
- Weekly report (`analysis/weekly-report-latest.md`)
- Anomalies (`data/anomalies.json`)
- Campaign health (`data/campaign-health.json`)
- Budget pacing (`data/budget-pacing.json`)
- Data quality (`data/data-quality.json`)

**Agenda (live):**
- KPI scorecard review (vs targets and vs prior week)
- Anomalies triage (real vs noise)
- Campaign health summary (what is broken / risky)
- Decide and assign actions (owners + deadlines)
- Confirm learning budget for the week (what we are willing to “pay to learn”)

**Output (same day):**
- Actions JSON updated (`data/weekly-actions.json`) and mirrored as a short bullet list in the report.

### Mid-week (Wed): Creative Performance Review (45–60 min)
Goal: convert early-week signal into creative direction and production priorities.

**Inputs:**
- Creative performance snapshot (last 3–7 days) segmented by geo/platform
- Fatigue check: frequency, CPM trend, CTR trend, CVR trend
- Top/bottom concepts and what to iterate

**Decisions:**
- Double-down concepts (new variations)
- Kill concepts (stop production)
- New hypotheses (angles/hooks/offer/LP message match)

**Output:**
- Creative brief batch for next production cycle + trafficking plan.

### Friday: Budget / Pacing / Scaling (45–60 min)
Goal: prevent surprises, re-allocate intelligently, lock next week’s plan.

**Inputs:**
- Pacing vs weekly/monthly budget
- Week-to-date CPA/CAC by geo/platform
- Saturation/fatigue and marginal efficiency

**Decisions:**
- Re-allocate budgets across geos/platforms
- Define next week’s scaling lane + learning lane
- Confirm stop-loss limits for weekend

**Output:**
- Budget plan + any “weekend rules” (what can auto-scale, what must be frozen).

### Always-on (async)
- Daily quick check (10 min): pacing + critical anomalies + platform disapprovals.
- Incident response: attribution breaks, fraud spikes, KYC breakages.

## 3) Decision Rules (how we pause/scale/kill)

Principle: **avoid endless debate**. Decisions are rule-based by default; exceptions require written rationale.

### 3.1 Pause Rules (protect downside)
Pause (or heavily reduce spend) when **any** is true, unless explicitly marked as learning budget:

- **Efficiency breach:**
  - Spend ≥ meaningful threshold (set per channel; default: $200 per ad set / $500 per campaign) AND
  - CPA/CAC is **> 1.35× target** over a 3-day window OR **> 1.50× target** over a 7-day window.
- **Conversion collapse:** CVR to activation drops **> 25% vs prior 7d baseline** with no matching tracking explanation.
- **Tracking suspicion:** clicks/impressions rising but downstream events flat (see Attribution Break rules).
- **Fraud suspicion:** abnormal click patterns / installs without post-install events / impossible geo/device mix.

What “pause” means:
- Immediate: cut budget by 80–100% or set to minimum viable spend for diagnostics.
- Document: add a note to `weekly-actions.json` with reason and evidence.

### 3.2 Scale Rules (earn the right to spend more)
Scale only when we have **signal quality** (enough volume) and **stable efficiency**.

Eligible to scale when **all** are true:
- CPA/CAC ≤ **0.90× target** (or ≥10% better than blended benchmark) over last 3 days.
- Delivery stable: no policy risk, no major learning limitations, no recent tracking gaps.
- Creative freshness: at least 1–2 new creatives in rotation or stable CTR/CVR without fatigue signs.

Scaling mechanics:
- Increase budgets **+15–25% per day** (avoid resets) until efficiency degrades.
- If scaling causes CPA/CAC to cross **1.05× target**, stop increasing and diagnose.

### 3.3 Dead Campaign Rules (stop wasting cycles)
A campaign/ad set is “dead” when it consistently fails to improve with reasonable iteration.

Kill (archive) when any is true:
- **3 iterations, no lift:** three distinct creative/targeting iterations and CPA/CAC still > 1.35× target.
- **No downstream quality:** installs/signups occur but activation/KYC pass rate is materially below baseline.
- **Structural mismatch:** audience/geo product constraints make conversion unlikely (e.g., KYC availability).

What “kill” means:
- Turn off + archive naming convention; summarize learning in the weekly report.
- Create a replacement test only if it’s a **new hypothesis**, not a re-skin.

### 3.4 Fraud Rules (act fast, assume it gets worse)
Trigger a fraud incident when any is true (Data Lead flags; Acquisition executes):

- Spike in clicks/impressions with flat conversions.
- CTR abnormally high with low session depth / no post-install events.
- New publisher placements show extreme volume + near-zero quality.
- Device/OS/geo mix shifts suddenly without a planned change.

Immediate actions (same day):
- Exclude suspicious placements/publishers.
- Tighten geo targeting and placement controls.
- Enable stricter attribution/reject rules if available.
- Add holdout: cap spend while investigating.

Follow-up (within 48h):
- Document in incident log section of weekly report.
- If confirmed, request refunds/credits where possible and update blocklists.

## 4) Artifacts (what must exist every week)

All artifacts are **versioned** and also have a “latest” pointer for quick access.

### 4.1 Weekly Report (Markdown)
- **Path:** `analysis/weekly-report-YYYY-MM-DD.md` + `analysis/weekly-report-latest.md`
- **Owner:** Data Lead
- **Contents:** KPI scorecard, insights, anomalies, health, actions, risks, next-week plan.

### 4.2 Actions JSON (source of truth for execution)
- **Path:** `data/weekly-actions.json`
- **Owner:** Acquisition Lead (with Data Lead writing initial recommendations)
- **Required fields (minimum):**
  - `week_start` (YYYY-MM-DD)
  - `items[]: {id, owner, channel, geo, scope, action, rationale, metric, threshold, due_date, status}`
  - `notes`

### 4.3 Campaign Health Snapshot
- **Path:** `data/campaign-health.json`
- **Owner:** Data Lead
- **Must cover:**
  - delivery status (active/limited/disapproved)
  - spend concentration risk
  - fatigue indicators
  - broken links/LP issues (if tracked)

### 4.4 Budget Pacing
- **Path:** `data/budget-pacing.json`
- **Owner:** Acquisition Lead (Data generates; Acquisition confirms)
- **Must cover:**
  - MTD spend vs plan
  - daily required spend to hit plan
  - overspend/underspend flags

### 4.5 Data Quality & Instrumentation Status
- **Path:** `data/data-quality.json`
- **Owner:** Data Lead
- **Must cover:**
  - freshness (last updated timestamps)
  - missing events / schema drift
  - reconciliation checks (AppsFlyer vs Amplitude/GA4 deltas)

## 5) Escalation Paths (when to wake people up)

### 5.1 KYC / Onboarding Bugs (Product escalation)
**Trigger:** a sudden activation/KYC pass-rate drop, or user reports, or funnel step conversion collapse.

- **Detection (Data Lead):**
  - flag in anomalies + quantify impact on CAC (e.g., “expected activations lost/day”).
- **Immediate action (Product Liaison):**
  - open a P0/P1 ticket with reproduction steps + affected geos/platforms + timestamps.
- **Mitigation (Acquisition Lead):**
  - reduce spend on affected segments; shift to unaffected geos/platforms if possible.
- **Resolution SLA:**
  - P0 (hard block): same day hotfix plan + hourly updates.
  - P1 (major degradation): fix within 48h or marketing re-allocates budget away.

### 5.2 Attribution Break / Tracking Incident (Data + Acquisition escalation)
**Trigger:** paid traffic continues but downstream events disappear or diverge sharply.

- **Detection signals:**
  - AppsFlyer installs up, Amplitude activations flat
  - GA4 sessions up, signups down unexpectedly
  - event freshness fails (stale timestamps)
- **Immediate action (Data Lead):**
  - declare “tracking incident” in the weekly report (and in Slack/issue tracker if used)
  - run reconciliation checklist (source vs downstream)
- **Mitigation (Acquisition Lead):**
  - cap spend to learning budget only until tracking is restored
- **Eng escalation (Product Liaison):**
  - instrumentation owners looped in; confirm latest release changes; rollback if needed.
